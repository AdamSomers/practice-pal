# v0.2.0 — April 19, 2026

## New Features
- **Technique practice items**: New chart category for Hanon exercises and finger drills. Supports label, description, and optional BPM, like scales.
- **BPM range**: Teachers can set a BPM range instead of a single value. Displays as "100-120 BPM" when both are filled.
- **Progress graph time range**: Selector on Goals & Progress chooses the window (1w, 2w, 1mo, 3mo, 6mo). Selection persists per studio. Longer ranges bucket by week instead of day.
- **Session pause**: Pause button on the practice timer. Per-studio "Allow session pausing" setting in Studio Settings (default on). Paused time is excluded from elapsed.
- **Goal editing**: Pencil icon on active goals opens the goal form pre-filled for editing.

## Bug Fixes
- **Reward Settings disappearing**: Clicking a category chip or custom reward action no longer causes the Reward Settings panel to vanish. The PATCH /studios response now includes the role, so canEdit stays true after update.
- **Can't delete a custom reward used in a goal**: The goal.custom_reward_id FK is now ON DELETE SET NULL. Goals keep their snapshotted reward title after the custom reward is deleted.
- **Completed goals are deletable**: Delete button now appears on completed goals.

## Polish
- **Confirm goal completion**: Marking a goal complete now asks first.
- **Confirm end session**: "End session" on the practice player asks before exiting.
- **Theory items simplified**: No more repetitions or modifiers for theory — just topic and description.
- **Description on chart cards**: Theory, technique, and Other items show their description line on the chart edit page (matches the practice player).
- **Left hand / Right hand modifiers**: "Hands separate" is replaced by explicit "Left hand" and "Right hand" modifier suggestions.

## Migration Notes
- Adds `progress_time_range` (text) and `allow_pausing` (boolean, default true) columns to `studios`.
- Adds `'technique'` to the `chart_category` enum.
- Changes `goals.custom_reward_id` FK to ON DELETE SET NULL.
- These migrations are applied via raw SQL at deploy time (drizzle-kit push doesn't work here because it wants to drop the connect-pg-simple `session` table).
